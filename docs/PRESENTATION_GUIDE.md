# QuantLab — 全栈量化投研系统

> 个人从零搭建的一站式 A 股量化平台：数据→因子→策略→AI→模拟交易

---

## 一、项目概述

**量化投研全链路平台**，覆盖 A 股数据采集、因子工程、策略开发、多智能体决策、模拟交易的完整闭环。

- **定位**：个人量化研究系统，不做实盘交易
- **数据规模**：1,369 万行日线（2010-2026），7 大维度（行情/研报/公告/龙虎榜/融资融券/资金流/分红）
- **技术栈**：Python 3.12, Pandas/NumPy/Parquet, PyTorch, OpenAI GPT-4, Docker
- **代码量**：后端 ~8,000 行，30+ 研究策略，6 个 AI Agent

### 设计要点

- **数据闭环**：所有数据本地化存储，不依赖外部 API 实时运行
- **引擎统一**：单一回测入口（`research_backtest.py`），29 个策略通过 PRESETS 注册一键调用
- **无未来信息**：周五收盘评分 → 下周一开盘执行，使用 `next_open` 而非 `close`
- **路径解耦**：全部通过 `.env` 配置，零硬编码

## 三、核心技术亮点

### 3.1 多智能体决策系统

**架构**：6 个 Specialist Agent + 1 个 CIO Coordinator

| Agent | 职责 | 方法论来源 |
|-------|------|-----------|
| 基本面 | 估值/成长/质量评估 | Graham & Dodd + McKinsey |
| 技术面 | 量价形态/趋势/支撑阻力 | Wyckoff + Market Profile |
| 情绪 | 市场情绪/逆向指标 | Kahneman 行为金融学 |
| 消息 | 事件驱动/新闻催化 | Soros Reflexivity |
| 宏观 | 经济周期/大类资产配置 | Dalio 经济机器 + 美林时钟 |
| **Coordinator** | **综合决策，3/5 一致性验证** | CIO 决策框架 |

**决策流程**：
1. 每个 Agent 输出 `AgentSignal`（类型/置信度/评分/理由）
2. Coordinator 按权重层级验证：宏观 > 基本面 > 技术面 > 消息面 > 情绪
3. **3/5 法则**：至少 3 个 Agent 方向一致才可操作
4. 宏观定仓位，基本面选股，技术面择时

> **面试话术**："这不是简单的信号拼接，而是模拟真实投研团队的决策流程。每个 Agent 有自己的分析框架，CIO 综合多个维度后做最终判断，权重视框架层级递减。"

### 3.2 因子工程体系

**因子库规模**：
- Alpha191：191 个 GTJA 量价因子（含因子缓存 49 万条记录）
- Alpha158：158 个 Qlib 风格因子（短线/中线适用）
- 自定义因子族：MR（均值回复）、流动性、动量、低波动等
- **因子审计**：逐因子 IC/RankIC → 分组收益 → 逐年稳定性 → 多段压力测试

**关键发现**：
- `mr_ma20_residual`：IC 稳定 -0.138（所有年份一致负向），适合做空对冲
- `alpha_132`：核心动量因子，IC 随时间衰减（-1.5%/yr），触发因子退化应对设计
- 逐年复核系统：任何策略必须在 2012-2026 完整区间通过，拒绝短样本投机

**因子开发规范**（系统内置）：
```
因子名称 / 数据来源 / 是否未来数据 / 是否 proxy /
适用行情 / 失效场景 / 回测区间 / IC/RankIC/分组收益/回撤
```

> **面试话术**："我们不只看因子 IC 大小，更关注逐年稳定性和失效模式。alpha132 的逐年衰减直接驱动了 V8 策略的权重衰减设计，这是从数据中发现问题的典型案例。"

### 3.3 V8 策略体系（核心成果）

**4 代迭代演进**：

| 版本 | 评分公式 | 收益 | 夏普 | 关键改进 |
|------|---------|------|------|---------|
| V4 | 纯 α132 | +1,562% | 0.94 | 单因子基准 |
| V6 | α132 + ret60 + 小盘 | +4,902% | — | 多因子组合 |
| V7 | V6 + 严格过滤 | +6,524% | 1.21 | 约束增强 |
| **V8** | **V7 + α132 衰减** | **+7,350%** | **1.23** | **因子退化应对** |

**V8 评分公式**：
```
Score = α132(衰减后).rank × 50%
      + ret60.rank × 30%
      + (-liquidity).rank × 20%
```

**α132 衰减机制**：
- 2018 年起每年衰减 1.5%：`α132_adj = α132 × max(0.1, 1 - 0.015 × years_since_2018)`
- 驱动因素：回测发现 α132 因子 IC 随市场有效性提升而下降
- 效果：衰减后 ret60（中期动量）逐渐成为主导，保持策略自适应能力

**参数设置**：top_n=5, retention_multiple=3, 大盘过滤, 最低价格 3 元, 最低上市 120 天

**压力测试结果**：
- 924 政策移除：收益降至 +5,066%（-31%），但策略仍然有效
- α132 双倍衰减：收益升至 +8,094%（动量替代 α132 成为主要 alpha）
- 自举尾部风险：10% 概率两年内 -350%，**必须做风险削减**

> **面试话术**："V8 的核心价值不是 7350% 这个数字，而是展示了我们如何系统性地解决因子退化问题——通过数据监控发现衰减、设计衰减机制、做多段压力测试验证。这个方法论可以复用到任何策略。"

### 3.4 回测引擎设计

**关键设计决策**：

| 决策项 | 选择 | 原因 |
|-------|------|------|
| 频率 | 周频 | 减少交易成本，避免日频过拟合 |
| 执行价格 | 下一交易日开盘 + 0.1% 滑点 | 消除收盘价未来信息 |
| 交易成本 | 万三佣金 + 印花税 + 滑点 | 真实市场条件 |
| 大盘择时 | MA20 < MA60 = 空仓 | 避免系统性下跌 |
| 停牌处理 | 无法卖出时继续持有 | 模拟真实流动性约束 |
| 持仓留存 | top_n × retention_multiple | 降低换手，避免频繁进出 |
| 候选池 | 按流动性取前 800 名 | 确保可交易性 |

**回测假设明确声明**（引擎自动输出）：
```
engine_version / signal_frequency / signal_timing / execution_timing /
execution_price / initial_equity / market_filter / universe_size / top_n /
retention_multiple
```

> **面试话术**："每个回测假设都明确记录在输出报告中。面试者经常问'你如何处理未来信息'——我的回答是：周五收盘产生信号，下一交易日开盘执行，使用 next_open 价格，这从根本上消除了未来信息泄漏。"

### 3.5 AI 预测模型

**三种模型内置**（`backend/models/`）：

- **LSTM**：基于序列的股价趋势预测
- **Transformer**：注意力机制的时序建模（参考 AlphaFold/TimeSformer 思路）
- **强化学习 (RL)**：交易决策优化，环境 = 市场 + 组合，奖励 = 夏普/收益

与因子策略的关系：模型输出作为 Agent 的信号之一，不独立上线。

### 3.6 风险控制体系

**三层过滤**：

| 层级 | 机制 | 效果 |
|------|------|------|
| 宏观 | 三路大盘验证（上证/中证500/全A等权） | 硬下跌空仓 |
| 个股 | 价格≥3/上市≥120天/成交额>0/排除ST | 排除垃圾股 |
| 策略 | 持仓留存/流动性候选池/行业上限 | 分散风险 |

**策略分级制度**（内置在 PRESETS 中）：
- `可研究`：V6/V7/Alpha191 研究版
- `可模拟`：V8（需风险削减）
- `禁止上线`：V9 (IC-adaptive)
- `验证未通过`：leader_relay_daily (-68.84%)

> **面试话术**："我们有一个策略分级系统，V9 虽然收益也不错(+3011%)，但 IC-adaptive 机制在压力测试中暴露了尾部风险，因此被标记为'禁止上线'，永远不会被推进到模拟交易。这不是技术问题，是风险文化问题。"

## 六、面试问答准备

### Q: 最大的技术挑战是什么？
**A**: Alpha191 因子在 numpy 2.4 + pandas 3.0 下的 C 扩展二进制兼容性问题。ta-cn 的 C 扩展在新版 numpy 下 SIGBUS 崩溃。解决方式：pyproject.toml 锁定 numpy<2.0 + pandas<2.3，Python 3.12 环境固定。

### Q: 如何避免过拟合？
**A**: (1) 逐年复核，拒绝短样本；(2) 多段压力测试（924 移除/双衰减/自举）；(3) 冻结验收期设计；(4) V9 虽然收益不错但尾部风险过高，直接禁止上线。

### Q: 实盘和回测差距在哪里？
**A**: 最大差异在流动性冲击。回测假设所有候选股都能按开盘价成交，但小盘代理策略买 3 万元的实际冲击成本可能比回测高 3-5 倍。这是 V8 限制 top_n=5 且只从流动性前 800 名选股的原因。

### Q: 为什么不用聚宽/米筐？
**A**: (1) 因子开发需要全量历史数据无限制访问；(2) 自定义 Agent 系统无法在平台上运行；(3) 数据管道完全自控，不依赖外部平台稳定性；(4) 但这个选择也意味着 -> 策略无法跨平台移植，所有 pipeline 需自己维护。
